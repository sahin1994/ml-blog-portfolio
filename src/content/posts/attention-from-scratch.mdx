Attention is, at heart, a weighted average. Each query row looks at every key,
scores the match with a dot product, and uses those scores to blend the values.
The whole operation fits in one line of math:

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^{\top}}{\sqrt{d_k}}\right) V
$$

The $1/\sqrt{d_k}$ factor keeps the dot products from growing with dimension —
without it, softmax saturates and gradients vanish.

## The scaled dot-product

Here it is in code. No autograd, no framework — just NumPy, so the mechanism has
nowhere to hide.

```python
import numpy as np

def softmax(x, axis=-1):
    x = x - x.max(axis=axis, keepdims=True)
    e = np.exp(x)
    return e / e.sum(axis=axis, keepdims=True)

def attention(Q, K, V):
    # scores: (n_queries, n_keys)
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)
    weights = softmax(scores)
    return weights @ V
```

<Callout type="warning" title="Watch the numerics">
Subtracting the row max before `np.exp` isn't cosmetic — skip it and large
scores overflow to `inf`. This is the single most common bug when people write
softmax by hand.
</Callout>

## Putting it together

That's the entire idea. Everything a transformer adds on top — multiple heads,
masking, positional information — is scaffolding around these nine lines. In the
